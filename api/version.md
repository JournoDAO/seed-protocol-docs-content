# Version

**Version** represents the versioning system in Seed Protocol. Content published to Seed maintains git-style history—every change, revision, and update is tracked and can be traced over time.

## Overview

The versioning system enables:

- **Auditable history** — Trace how content evolved from draft through publication
- **Fork and merge** — Work from specific versions and merge improvements back
- **Immutable timestamps** — Each version is recorded with permanent timestamps

## Use cases

- Writers tracking drafts and revisions
- Researchers maintaining versioned datasets
- Journalists creating auditable chains from research to publication
